## 实验执行总体规划

整个实验可划分为四个开发配置阶段和一个测试验证阶段。我们将使用 `docker-compose` 来统一管理所有服务，确保环境的隔离与一键启动 。



### 阶段一：数据库层 (MySQL) 配置

首先需要准备数据库的初始化脚本，以便在 Docker 启动 MySQL 时自动建库建表并注入数据。

1. 在项目根目录创建一个 `mysql` 文件夹。

2. 在其中新建 `init.sql` 文件，写入以下内容以满足实验要求：

   - 创建名为 `mall_db` 的数据库 。

     

     

   - 创建包含 `id`, `name`, `price` 字段的 `product` 表 。

     

     

   - 插入 5 条预置的测试数据 。

     

     

SQL

```
CREATE DATABASE IF NOT EXISTS mall_db DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
USE mall_db;

CREATE TABLE product (
    id INT AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(100) NOT NULL,
    price DECIMAL(10, 2) NOT NULL
);

INSERT INTO product (name, price) VALUES 
('机械键盘', 499.00), ('电竞鼠标', 299.00), ('27寸显示器', 1599.00),
('降噪耳机', 899.00), ('人体工学椅', 1299.00);
```

### 阶段二：业务服务层 (Spring Boot) 开发

此阶段需要编写一个提供 RESTful API 的微服务，并将其容器化 。



1. 

   **编写 API 接口**：创建一个 `ProductController`，提供 `GET /api/products` 接口 。

   

   

2. 

   **获取实例标识**：在返回的 JSON 数据中，必须额外增加 `server_port` 或 `instance_id` 字段 。可以通过注入 `Environment` 对象来动态获取当前运行的端口：

   

   

   Java

   ```
   @RestController
   @RequestMapping("/api")
   public class ProductController {
       @Autowired
       private Environment environment;
       // 假设注入了 productService 获取数据库数据
   
       @GetMapping("/products")
       public Map<String, Object> getProducts() {
           List<Product> products = productService.getAllProducts();
           String port = environment.getProperty("local.server.port");
   
           Map<String, Object> response = new HashMap<>();
           response.put("server_port", port); // 清楚展示请求落在哪台实例上
           response.put("data", products);
           return response;
       }
   }
   ```

3. 

   **打包为 Docker 镜像** ：在 Spring Boot 项目根目录编写 `Dockerfile`：

   

   

   Dockerfile

   ```
   FROM openjdk:17-jdk-slim
   WORKDIR /app
   COPY target/mall-app.jar app.jar
   ENTRYPOINT ["java", "-jar", "app.jar"]
   ```

### 阶段三：负载均衡层 (Nginx) 配置

Nginx 作为统一入口，负责将流量分发给后端的应用集群 。



1. 在项目根目录创建一个 `nginx` 文件夹，新建 `nginx.conf` 文件 。

   

   

2. 使用 `upstream` 块定义后端服务器组，并配置健康检查与权重 ：

   

   

   Nginx

   ```
   events {
       worker_connections 1024;
   }
   
   http {
       upstream backend_servers {
           # 默认轮询策略，配置权重模拟服务器性能差异
           # 配置 max_fails 和 fail_timeout 实现健康检查和故障自动剔除
           server app1:8080 weight=2 max_fails=2 fail_timeout=10s; 
           server app2:8080 weight=1 max_fails=2 fail_timeout=10s;
       }
   
       server {
           listen 80;
   
           location / {
               proxy_pass http://backend_servers; # 指向服务器组
               proxy_set_header Host $host;
               proxy_set_header X-Real-IP $remote_addr;
           }
       }
   }
   ```

### 阶段四：容器编排 (Docker Compose)

使用 `docker-compose.yml` 文件一键启动 1 个 Nginx、2 个完全相同的 Spring Boot 实例和 1 个 MySQL 容器 。



在项目根目录创建 `docker-compose.yml`：

YAML

```
version: '3.8'

services:
  mysql:
    image: mysql:8.0
    environment:
      MYSQL_ROOT_PASSWORD: root
    volumes:
      - ./mysql/init.sql:/docker-entrypoint-initdb.d/init.sql
    ports:
      - "3306:3306"

  app1:
    build: ./spring-boot-app # 假设 Spring Boot 代码在此目录
    environment:
      - SPRING_DATASOURCE_URL=jdbc:mysql://mysql:3306/mall_db?useSSL=false&allowPublicKeyRetrieval=true
      - SPRING_DATASOURCE_USERNAME=root
      - SPRING_DATASOURCE_PASSWORD=root
      - SERVER_PORT=8080
    depends_on:
      - mysql

  app2:
    build: ./spring-boot-app
    environment:
      - SPRING_DATASOURCE_URL=jdbc:mysql://mysql:3306/mall_db?useSSL=false&allowPublicKeyRetrieval=true
      - SPRING_DATASOURCE_USERNAME=root
      - SPRING_DATASOURCE_PASSWORD=root
      - SERVER_PORT=8080
    depends_on:
      - mysql

  nginx:
    image: nginx:latest
    volumes:
      - ./nginx/nginx.conf:/etc/nginx/nginx.conf
    ports:
      - "80:80"
    depends_on:
      - app1
      - app2
```
