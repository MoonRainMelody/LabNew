# 基于 ShardingSphere-JDBC 的 MySQL 读写分离项目展示

## 📋 项目概述

本项目使用 **Apache ShardingSphere-JDBC 5.3.0** 实现了 **MySQL 8.0 主从数据库的读写分离**功能。

### 核心技术栈
- **Spring Boot**: 2.7.18 (LTS长期支持版本)
- **ShardingSphere-JDBC**: 5.3.0 (分布式数据库中间件)
- **MySQL**: 8.0 (Docker容器化部署)
- **主从架构**: 1主1从 (Master-Slave复制)
- **开发工具**: Maven + Git

---

## 🎯 读写分离证明

### 1. Docker MySQL 主从环境

**主库 (Master)**: `localhost:3306`
- 负责所有写操作 (INSERT/UPDATE/DELETE)
- 数据源名称: `master`

**从库 (Slave)**: `localhost:3307`
- 负责所有读操作 (SELECT)
- 数据源名称: `slave`

### 2. ShardingSphere 配置

**配置文件**: `src/main/resources/shardingsphere-config.yaml`

```yaml
# 读写分离规则配置
rules:
- !READWRITE_SPLITTING
  dataSources:
    readwrite_ds:
      staticStrategy:
        writeDataSourceName: master    # 写→主库
        readDataSourceNames:
          - slave                        # 读→从库
```

### 3. 运行日志证明

**写操作日志** (创建用户):
```log
[ShardingSphere] Actual SQL: master ::: INSERT INTO users ...
                            ↑
                        写操作路由到主库
```

**读操作日志** (查询用户):
```log
[ShardingSphere] Actual SQL: slave ::: SELECT * FROM users ...
                            ↑
                        读操作路由到从库
```

## 🧪 功能演示

### 演示步骤

#### 步骤1: 查看Docker主从环境
```bash
docker ps
```
**预期结果**: 看到两个MySQL容器
- `mysql-master` (端口3306)
- `mysql-slave` (端口3307)

#### 步骤2: 查看主库数据
```bash
docker exec mysql-master mysql -uroot -proot123 -e "USE practice_db; SELECT * FROM users;"
```

#### 步骤3: 查看从库数据
```bash
docker exec mysql-slave mysql -uroot -proot123 -e "USE practice_db; SELECT * FROM users;"
```

#### 步骤4: 验证主从同步
```bash
docker exec mysql-slave mysql -uroot -proot123 -e "SHOW SLAVE STATUS\G" | grep "Slave_IO_Running"
```
**预期结果**: `Slave_IO_Running: Yes`

#### 步骤5: 测试写操作 (创建用户)
```bash
curl -X POST http://localhost:8080/api/users \
  -H "Content-Type: application/json" \
  -d '{"username":"test_user","email":"test@example.com","phone":"18888888888"}'
```

#### 步骤6: 验证数据写入主库
```bash
docker exec mysql-master mysql -uroot -proot123 -e "USE practice_db; SELECT * FROM users WHERE username='test_user';"
```
**预期结果**: 主库中有新创建的用户

#### 步骤7: 验证数据同步到从库
```bash
docker exec mysql-slave mysql -uroot -proot123 -e "USE practice_db; SELECT * FROM users WHERE username='test_user';"
```
**预期结果**: 从库中也有相同的用户 (证明主从同步正常)

#### 步骤8: 测试读操作 (查询用户)
```bash
curl http://localhost:8080/api/users
```
**预期结果**: 返回所有用户列表，数据来自从库

## 🔍 关键技术点说明

### 1. 为什么使用 ShardingSphere-JDBC？
- **应用层路由**: 无需修改业务代码
- **透明化**: 对业务层完全透明
- **轻量级**: JDBC层面的增强，无需代理

### 2. 读写分离策略
- **写操作**: 全部路由到主库 (master)
- **读操作**: 全部路由到从库 (slave)
- **事务处理**: 事务内的所有操作都走主库

### 3. 主从同步机制
- **MySQL Binlog**: 基于二进制日志的异步复制
- **GTID模式**: 全局事务ID，确保数据一致性
- **实时同步**: 毫秒级延迟
